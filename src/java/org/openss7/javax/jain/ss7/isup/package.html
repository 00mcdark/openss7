This package provides the OpenSS7 Project implementations of the vendor-specific components of the JAIN ISUP API. In particular, it provides the JAIN ISUP stack and provider implementation classes. The JainIsupStackImpl class implements the vendor-specification stack implementation class of the JAIN ISUP API as provided by the JainIsupStack interface. Instances of this class are obtained using the JainSS7Factory object with a path name of org.openss7 and an object name of javax.jain.ss7.isup.JainIsupStackImpl.

Each JainIsupStackImpl object created in this fashion represents a single, unique, local network and signalling point code (network appearance). For additional local signalling point codes, an additional JainIsupStackImpl object should be acquired and appropriately configured using the setNetworkIndicator(), setSignalingPointCode() or setVendorName() methods. In the OpenSS7 implementation of the JAIN ISUP stack vendor class, each instance of the vendor class corresponds to a local binding of the ISUP stack.

Each JainIsupStackImpl can be used to acquire new instances of the JainIsupProviderImpl object using the createProvider() method. The vendor-specific provider class happens to be called org.openss7.javax.jain.ss7.isup.JainIsupProviderImpl, but this is not mandated by the JAIN ISUP API specification as it is for the JainIsupStackImpl object. In the OpenSS7 implementation of the JAIN ISUP provider vendor class, each instance of the vendor class corresponds to a locallly bound ISUP stack stream.

User classes that implement the JainIsupListener interface can register to send and receive events with a JainIsupProviderImpl object for a given ISUP address. The ISUP address consists of the local signalling point code to which the provider is already bound, the remote signalling point code (of the remote switch), and the circuit identification code range identifying circuits between the two signalling points.
