This package contains Java Call Control (JCC) API interfaces.

The following tables give an overview of the side effects of the JccCall and JccConnection methods. These tables are here given for convenience: for details, see the method description.

Table 1: JccConnection state acronym mapping
S1=IDLE S4=ADDRESS_ANALYZE S7=CONNECTED
S2=AUTHORIZE_CALL_ATTEMPT S5=CALL_DELIVERY S8=DISCONNECTED
S3=ADDRESS_COLLECT S6=ALERTING S9=FAILED

Table 1 gives a number of JccConnection states: S1 to S9. This shorthand is introducted to reduce the size of the columns in subsequent tables.

The answer() method can only be successfully invoked on a Terminating Connection; invoking it on an Originating Connection will not cause a transition to the CONNECTED state.

Table 2 contains a list of methods and 3 different types of columns. A "Yes" in the first column, labelled "Implicit processing resume?", denotes that invoking the method causes blocked processing to resume. A "Yes" in the second column, labeled "Invoke when not blocked?", denotes a method can be invoked if the processing is not blocked. The remaining columns list al states where a method can be successfully invoked in the state and the resulting state.
